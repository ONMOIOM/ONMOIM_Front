# 0205 메인 / 분석 관련 정리

현재 상황과 해야 할 작업을 정리한 문서입니다.

---

## 1. 검색 관련

- **상황**: 검색 기능 추가? 예정?.
- **해야 할 것**: 검색 기능을 넣는다면 **디바운싱(debouncing)** 으로 구현할 것.

---

## 2. 문의하기 페이지 관련

- **상황**: 이전에 설명을 들었으나 기억이 잘 나지 않음.
- **해야 할 것**: 다시 물어보기.

---

## 3. 분석 페이지

- **상황**: 분석 페이지 미구현.
- **해야 할 것**: 분석 페이지 구현해야 함.

---

## 4. 행사 정보 보기 페이지

- **상황**: 행사 상세/정보 보기 페이지 없음.
- **해야 할 것**: 행사 정보 보기 페이지 만들어야 함.

---

## 5. 알람 모달창

- **상황**: 알람 모달은 목록이 많아질 수 있음.
- **해야 할 것**: **무한 스크롤(infinite scroll)** 로 구현할 수 있음.

---

## 6. 행사 리스트 & 같은 행사에 참여한 분들

- **상황**: 행사 리스트, 같은 행사에 참여한 분들 수가 많아질 수 있음.
- **해야 할 것**:
  - 수가 많아지면 **스크롤** 및 **`<` `>` 버튼**으로 대응.
  - 많은 수에 대비한 UI/UX 구현.

---

### 댓글 관련

- **댓글 작성** API 추가.
- **댓글 삭제** API 추가.

### 프로필 페이지(이메일 변경)

- 이메일 변경 시 **이메일 중복 유효성 검사**를 위해, 이 웹에 회원가입한 **모든 사용자의 이메일을 조회할 수 있는 API** 추가.

### 분석 페이지(통계값 획득)

- 통계값 획득 API는 **"내가 주최한 모든 행사의 통계"를 한눈에 보는 페이지**용으로 사용.
- API 응답은 **배열 형태**가 되어야 함.

### 분석 페이지(주최한 행사 목록)

- 행사 분석 페이지에서 자신이 **"주최한 행사"**를 알기 위해 다음 중 하나 필요:
  - **"내가 주최한 행사 목록" 전용 API** 추가.
  - 또는 **GET /api/v1/users/events** 에 쿼리 파라미 또는 응답 필드(예: **isHost: true**)를 넣어, 주최한 행사만 구분 가능하게 할 것.
